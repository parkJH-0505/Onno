# Part 1: 제품 전략 (Why)

## 1-1. 제품 비전 (Product Vision)

### Vision Statement

> **"AI가 함께하는 모든 대화를 더 깊고, 더 생산적으로 만든다"**

Onno는 단순한 회의 도구가 아닙니다.
**실시간 대화 인텔리전스**를 통해 사람들이 더 나은 질문을 하고, 더 깊은 인사이트를 얻으며, 더 빠르게 실행할 수 있도록 돕는 **AI 대화 파트너**입니다.

### Mission

**"회의 중 놓치는 것을 없애다"**

- 놓친 질문 → AI가 실시간 제안
- 잊은 맥락 → AI가 자동 상기
- 수작업 업무 → AI가 자동 연결

### 3-5년 비전

```
Year 1 (2025):
VC/AC/CB를 위한 실시간 질문 제안 플랫폼
→ "회의 중 AI가 질문 제안하는 유일한 서비스"

Year 2 (2026):
모든 지식 노동자를 위한 대화 인텔리전스 플랫폼
→ 세일즈, 컨설팅, 채용, 교육 등 확장

Year 3-5 (2027-2029):
AI Analyst-as-a-Service
→ "AI가 회의 준비, 진행, 후속 조치까지 완전 자동화"
→ Service-as-Software 모델로 전환
```

## 1-2. 문제 정의 (Problem Statement)

### 핵심 문제

**"인간은 실시간 대화 중에 듣기·분석·기억·질문 생성을 동시에 할 수 없다"**

#### 인지 부하 분석

```
실시간 대화 중 필요한 4가지 인지 작업:

1. 듣기 (Listening)          → 60% 에너지
   상대방 발언 이해, 의도 파악

2. 분석 (Analyzing)          → 20% 에너지
   논리 허점, 누락 정보, 리스크 감지

3. 기억 (Recalling)          → 10% 에너지
   과거 대화, 도메인 지식, 벤치마크

4. 생성 (Generating)         → 10% 에너지
   다음 질문 구성, 후속 조치 계획

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
결과: 90%를 "듣기"에 쓰고, 10%만 나머지에 할애
→ 질문을 '알지만' '던지지 못하는' 상황 발생
```

### 정량적 Pain Point (고객 인터뷰 기반)

**VC 심사역 (N=15명)**
- 87%: "중요한 질문을 놓친 경험이 자주 있다"
- 73%: "회의 후 '아, 그거 물어볼 걸' 하고 후회한다"
- 60%: "체크리스트를 만들어도 회의 중엔 안 본다"

**AC 매니저 (N=10명)**
- 90%: "같은 팀에 같은 설명을 반복한다"
- 80%: "여러 팀 관리할 때 맥락을 혼동한다"
- 70%: "과거 회의 내용을 찾느라 시간 낭비한다"

**Company Builder (N=8명)**
- 100%: "사용자 인터뷰 후 검증이 미흡하다"
- 75%: "내부 편향을 깨는 질문을 못한다"
- 63%: "회의 내용을 보고서로 정리하는 게 귀찮다"

## 1-3. 타겟 고객 (Target Customers)

### Primary Target (MVP - Month 0-6)

**1. VC/PE 투자 심사역**
- **규모**: 한국 ~200명, 미국 ~5,000명
- **특징**:
  - "질문 잘해야 성과 나는" 직군
  - Deal flow 많고 시간 부족
  - 체크리스트 있지만 회의 중 활용 못함
- **Pain**: Question Timing Gap, Stage Blindness
- **지불 의향**: $20-30/month
- **획득 경로**: LinkedIn, VC 커뮤니티, 직접 아웃리치

**2. Accelerator 프로그램 매니저**
- **규모**: 한국 ~50명, 미국 ~500명
- **특징**:
  - 동시에 10-20개 팀 관리
  - 반복 설명 많음 (CAC, LTV 등)
  - 팀별 맥락 혼동
- **Pain**: Teaching Overload, Cohort Tracking Gap
- **지불 의향**: $15-25/month (회사 예산)
- **획득 경로**: AC 네트워크, 파트너십

### Secondary Target (Month 7-12)

**3. Company Builder / Venture Studio PO**
- **규모**: 한국 ~30명, 미국 ~200명
- **특징**:
  - 사용자 인터뷰 빈번
  - 가설 검증 중요
  - 내부 보고 많음
- **Pain**: Echo Chamber, Validation Rigor Gap
- **지불 의향**: $25-35/month

**4. B2B 세일즈 (AE, SDR)**
- **규모**: 글로벌 ~100,000명+
- **특징**:
  - 고객 미팅 빈번
  - CRM 업데이트 중요
  - 후속 조치 누락 많음
- **Pain**: Follow-up Weakness, CRM Manual Entry
- **지불 의향**: $15-20/month

### Tertiary Target (Year 2+)

**5. 컨설턴트 (전략, 마케팅, 조직)**
**6. HR/채용 담당자 (인터뷰)**
**7. 교육자 (멘토링, 코칭)**

### 경쟁 우위 (Competitive Advantage)

#### 1. During-the-fact 실시간 질문 제안 (Unique)

**기존**: 회의 '후' 요약·분석
**Onno**: 회의 '중' 질문 제안

→ **유일무이한 가치 제안**

#### 2. Vertical 깊이 (Defensible)

**기존**: 범용 (모든 회의)
**Onno**: VC/AC 특화 (투자 심사, 멘토링)

→ **도메인 지식, 체크리스트, 벤치마크 차별화**

#### 3. 개인화 (Sticky)

**기존**: 모든 사용자에게 동일
**Onno**: 나만의 Onno (학습, 레벨업)

→ **높은 전환 비용, 브랜드 충성도**

#### 4. 한국 시장 최적화 (Local Advantage)

**기존**: 영어 중심, PIPA 미준수
**Onno**: 한국어 최적화, HWP 파싱, PIPA 준수

→ **한국 VC/AC 시장 선점**

**Onno가 쌓을 수 있는 방어막**:

1. **데이터 모트** (6개월+)

### Unit Economics

```
고객 획득 비용 (CAC):
- Organic (70%): $0 (바이럴, 콘텐츠)
- Paid (30%): $50 (LinkedIn Ads, 컨퍼런스)
- Blended CAC: $15

생애 가치 (LTV):
- ARPU: $20/month
- 예상 Churn: 5%/month
- 예상 LTV: $20 × (1/0.05) = $400

LTV:CAC = $400 / $15 = 26.7
→ 건강한 SaaS (목표: 3:1 이상)

Payback Period:
$15 / $20 = 0.75개월
→ 매우 짧음 (목표: <12개월)
```

#### FR-002-3: AI 질문 제안 (Real-time Question Suggestion) (P0)

**기능**:
- 실시간 전사 텍스트 분석
- 대화 맥락 이해 (대화 단계 인식)
- 정보 누락 감지
- 우선순위별 질문 생성
  - 🔴 Critical: 지금 꼭 물어봐야 함
  - 🟡 Important: 추가 확인 권장
  - 🟢 Follow-up: 나중에 물어도 됨

**알고리즘**:
```
입력: 실시간 전사 텍스트 (마지막 N턴)

1. 대화 단계 인식 (LLM Classifier)
   - Introduction (자기소개)
   - Problem (문제 정의)
   - Solution (솔루션 설명)
   - Business Model (비즈니스 모델)
   - Financials (재무 지표)
   - Team (팀 구성)
   - Ask (요청 사항)
   - Close (마무리)

2. 정보 누락 감지 (Rule-based + LLM)
   예: "CAC $30" 언급 but "LTV" 미언급
   → Critical: "LTV는 얼마인가요?"

3. 도메인 체크리스트 매칭
   - 선택된 체크리스트 (예: Series A 필수 질문 20개)
   - 아직 확인 안 된 항목 추출
   - 현재 대화 단계와 관련성 높은 것 우선

4. 우선순위 라벨링
   - Critical: 단계별 필수 항목 + 모순 발견
   - Important: 체크리스트 미확인 항목
   - Follow-up: 심화 질문

5. 질문 생성 (LLM Prompt Engineering)
   Prompt:
   "대화 맥락을 고려하여, 다음 질문을 자연스러운 한국어로 생성하세요.
    - 맥락: {transcript}
    - 누락 정보: LTV
    - 질문 의도: CAC:LTV 비율 확인
    - 출력: '생애 가치(LTV)는 얼마인가요? CAC와 비교했을 때 수익성을 판단하고 싶습니다.'"

출력: 질문 카드 (우선순위, 텍스트, 이유)
```

**LLM Prompt 예시**:
```
System:
당신은 VC 투자 심사 전문가입니다.
현재 대화를 분석하여 심사역이 물어봐야 할 질문을 제안하세요.

User:
[대화 전사]
창업가: "저희 CAC는 $30입니다. 최근 3개월 평균이고, 주로 Google Ads를 통해 고객을 확보하고 있습니다."

심사역: "좋네요. 그럼 채널별로..."

[체크리스트]
- [ ] LTV (생애 가치) 확인
- [ ] CAC:LTV 비율 확인
- [ ] Churn Rate 확인
- [ ] 채널 다각화 전략

[지시사항]
1. 현재 대화 단계: Financials
2. 언급된 지표: CAC
3. 누락된 지표: LTV, Churn Rate
4. 지금 물어보기 좋은 질문 3개를 우선순위별로 제안하세요.
