## 1. 사업 아이템 개요

### 1-1. 한 줄 정의

**Real-Time Conversation Intelligence 'Onno'(온노)**는

회의·미팅에서 오가는 대화를 **실시간으로 이해**하고,
적절한 **질문·인사이트·맥락 설명**을 제안하며,
필요 시 CRM·PM·문서·이메일 등 기존 시스템으로 자연스럽게 연결해 주는
**실시간 대화 인텔리전스(AI 코파일럿)**입니다.

### 1-2. 서비스 포지션

기존 AI 회의 솔루션처럼 **"녹취·요약"**에 머무는 도구가 아니라,

**회의 진행 중(During-the-fact)**에
"무엇을 물어봐야 하는지, 어떻게 해석해야 하는지"를 도와주는
**대화 중심(Conversation-first)** 솔루션입니다.

#### 핵심 차별점 3가지

| 구분 | 기존 솔루션 | Onno |
|-----|-----------|------|
| **시점** | After-the-fact<br>(회의 후 요약) | **During-the-fact**<br>(회의 중 실시간 제안) |
| **가치** | 기록(System of Record) | **실행(System of Action)**<br>질문·의사결정 지원 |
| **차별화** | 범용 기능<br>(빅테크 플랫폼과 경쟁) | **도메인 특화 + 개인화**<br>(데이터 모트 구축) |

또한 사용할수록 개인·팀에 특화되어 성장하는 **'나만의 온노' 경험**을 제공하여
장기적인 서비스 충성도와 **데이터 해자(Data Moat)**를 동시에 구축합니다.

### 1-3. 시장 기회

#### 글로벌 회의 솔루션 시장 규모

- **2024년**: $4.5B
- **2030년 예상**: $15B+ (CAGR 22%)
- **한국 시장**: $200M+ (2024년 기준)

#### 한국 타겟 시장 (TAM/SAM/SOM)

```
TAM (Total Addressable Market)
└─ 한국 지식 노동자 전체: 500만 명
   └─ 잠재 시장 규모: $1.2B/year

SAM (Serviceable Addressable Market)
└─ 고관여 회의 직군: 50만 명
   (VC/AC, 컨설팅, B2B 세일즈, 마케팅 등)
   └─ 실질 시장 규모: $120M/year

SOM (Serviceable Obtainable Market)
└─ 초기 3년 목표: 1만 명
   └─ 목표 매출: $2.4M/year (Year 3)
```

## 2. 개발 필요성 및 문제 정의

### 2-1. 기존 회의 솔루션의 한계

현재 회의/미팅 AI 솔루션 대부분은
**"녹음 → 전사 → 요약 → 할 일 추출"**에 초점을 맞추고 있습니다.

#### Sherlocking 위기

Microsoft(Teams Copilot), Zoom(AI Companion), Google(Gemini in Meet) 등
빅테크 플랫폼들이 이미 이 기능을 **플랫폼 기본 기능(무료 혹은 번들)**로 제공하기 시작했습니다.

**결과**:
- "회의를 요약해 줍니다" 수준의 범용 기능으로는 별도 서비스 도입 유인 약화
- 다수의 AI 회의 스타트업이 플랫폼에 흡수·대체될 위험

### 2-2. 현장의 근본적인 Pain Point

VC/PE, 스타트업 창업자, 컴퍼니빌더, 컨설턴트, 마케팅 에이전시 등
**"말(대화)로 의사결정·합의를 만드는 직군"**에서는 다음과 같은 문제가 반복됩니다.

#### Pain Point 1: 좋은 질문의 부재

**문제**:
- 중요한 리스크/핵심 지표를 묻지 못한 채 미팅이 끝남
- 재회의·추가 검토가 반복됨

**정량적 데이터** (자체 인터뷰 N=33명):
- VC 심사역 87%: "중요한 질문을 놓친 경험이 자주 있다"
- AC 매니저 90%: "같은 팀에 같은 설명을 반복한다"
- Company Builder 100%: "사용자 인터뷰 후 검증이 미흡하다"

#### Pain Point 2: 의사결정의 재작업(Rework)

**문제**:
- 1차 회의에서 빠진 질문·논점을 2차·3차 회의에서 다시 다룸
- 시간·비용 과도 낭비

**비용 추정**:
- VC 1건당 평균 3-4회 미팅 (업계 평균 2-3회 대비 +50%)
- 재회의 비용: 시간당 $200 × 2시간 × 1.5회 = **$600/딜 추가 비용**

#### Pain Point 3: 주니어 인력의 Ramp-up 문제

**문제**:
- 주니어 심사역/컨설턴트/AE는 "어떤 관점에서 어떤 질문을 해야 하는지" 프레임 부족
- 시니어에게 의존, 효율 낮은 온보딩 반복

**결과**:
- 주니어 독립 업무 수행까지 평균 **6-12개월** 소요
- 교육 비용: 1인당 $10K-20K

#### Pain Point 4: '질문'과 '대화의 질'이 성과와 직결

**산업 특성**:
- 투자 검토, 전략 프로젝트, 캠페인 설정, 세일즈 미팅 등
- 한 번의 잘못된 질문·판단 → 수억~수십억 규모 손실 가능

### 2-3. 핵심 인사이트

> 현재 회의 도구들이 해결하는 것: **"어떻게 잘 기록할 것인가"**
>
> 실제 현장에서 가장 크고 비싼 문제: **"그 순간에 무엇을 묻고, 어떻게 생각할 것인가"**

**→ Onno는 후자를 해결합니다.**

## 3. 핵심 기능 및 서비스 구조

### 3-1. 핵심 기능 3축

Onno의 핵심은 세 가지 축으로 구성됩니다.

```
┌────────────────────────────────────────────────────┐
│  Axis 1: 실시간 질문·인사이트·맥락 제공            │
│  (Real-time Conversation Intelligence)            │
│                                                    │
│  - 회의 중 질문 추천                                │
│  - 도메인 특화 인사이트                             │
│  - 실시간 개념 해설                                 │
└────────────────────────────────────────────────────┘
                      ↓
┌────────────────────────────────────────────────────┐
│  Axis 2: 워크플로우 브리지                          │
│  (Workflow Bridge)                                │
│                                                    │
│  - CRM·PM·문서·메일 연동                           │
│  - 실행까지의 마찰 최소화                           │
└────────────────────────────────────────────────────┘
                      ↓
┌────────────────────────────────────────────────────┐
│  Axis 3: 나만의 온노 (개인화)                       │
│  (Personalized & Level-up Experience)            │
│                                                    │
│  - 사용 패턴 학습                                   │
│  - 레벨 시스템 (Lv.1-5)                            │
│  - 페르소나 모드 (4가지)                            │
│  - 데이터 모트 구축                                 │
└────────────────────────────────────────────────────┘
```

### 3-2. Axis 1: 실시간 질문 추천 기능

#### 기능 설명

회의 중 발언을 실시간 분석하여:

1. **현재 논의 단계 인식**
   - 문제 정의, 솔루션 설명, 비즈니스 모델, 재무 지표, 팀 구성, 마무리 등

2. **정보량·리스크 요소 파악**
   - 언급된 지표 vs 누락된 지표
   - 논리적 모순, 검증 필요 사항

3. **적절한 질문 3-5개 실시간 제안**
   - 우선순위별 라벨링 (Critical, Important, Follow-up)
   - 질문 이유 설명 포함

#### 실제 사용 예시 (VC–스타트업 미팅)

```
창업가: "저희 CAC는 $30입니다. 최근 3개월 평균이고,
        주로 Google Ads를 통해 고객을 확보하고 있습니다."

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Onno 실시간 제안:

🔴 Critical 질문
"LTV(생애 가치)는 얼마인가요?"

💬 이유: CAC만으로는 수익성 판단이 어렵습니다.
        LTV:CAC 비율이 3:1 이상이어야 건강한 비즈니스입니다.

[사용하기] [나중에]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🟡 Important 질문
"채널별 CAC 차이는 어떻게 되나요?"

💬 이유: 특정 채널(Google Ads) 의존도가 높을 경우
        채널 다각화 전략 확인이 필요합니다.

[사용하기] [나중에]
```

#### 효과

- 주니어 심사역도 **시니어급 질문 프레임** 즉시 활용
- 리스크·검증 누락 방지 → **재회의 50% 감소** (목표)
- 의사결정 품질 향상 → **잘못된 투자 판단 30% 감소** (목표)

### 3-3. Axis 1: 도메인 특화 인사이트 제공

#### 기능 설명

회의 중 언급된 지표(MRR, 성장률, CAC, LTV, ROAS 등)를 자동 추출하여:

1. **벤치마크 비교**
   - 사전 구축된 도메인별 기준과 자동 비교
   - "SaaS 평균", "동종 업계 상위 30%", "시리즈 A 평균" 등

2. **실시간 인사이트 제공**
   - 높음/낮음/적정 판단
   - 추가 질문 필요 여부 제안

#### 예시

```
언급된 지표: CAC $30, MRR $50K

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Onno 인사이트:

📊 벤치마크
CAC $30 = SaaS 평균 수준 (평균 $30-50)
MRR $50K = Seed 상위 30% (평균 $30K)

⚠️ 리스크 감지
LTV 미언급 → 수익성 불확실
→ 추가 확인 필요
```

### 3-4. Axis 1: 실시간 맥락 설명 및 개념 해설

#### 기능 설명

미팅 중 등장하는 **전문 용어·개념**을 화면 옆에서 간단히 설명:

- 재무·지표 용어
- 스타트업·투자·마케팅·컨설팅 도메인 개념
- 산업·시장 관련 용어

#### 예시

```
창업가: "우리 코호트 분석 결과 M1 리텐션이 80%입니다."

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Onno 개념 해설:

📚 용어 설명
코호트 분석(Cohort Analysis)
동일 시기에 가입한 사용자 그룹을 추적하여
시간에 따른 행동 변화를 분석하는 방법

M1 리텐션(Month 1 Retention)
가입 후 1개월 뒤에도 서비스를 사용하는 비율
SaaS 평균: 70-75%
```

#### 효과

- 회의 중 "이 용어를 몰라서 질문도 못 하는" 상황 방지
- 주니어/비전문가도 **실시간 이해도 향상** → 대화 참여 촉진

### 3-5. Axis 2: 워크플로우 브리지 (CRM·PM 연동)

#### 기능 설명

대화에서 도출된 **결정·할 일**을 도메인별 워크플로우에 맞춰 기존 시스템과 자동 연계:

| 도메인 | 회의 유형 | 자동 연계 예시 |
|-------|---------|--------------|
| **VC/AC** | 투자 심사 | • CRM(Deal) 상태 업데이트<br>• DD 체크리스트 초안 생성<br>• 후속 미팅 일정 제안 |
| **스타트업** | IR 미팅 | • 후속 자료 요청 메일 초안<br>• 내부 회의 캘린더 등록 |
| **마케팅** | 캠페인 회의 | • Asana/ClickUp 태스크 분배<br>• 브리프 문서 템플릿 생성 |
| **컨설팅** | 클라이언트 미팅 | • 제안서 초안 생성<br>• 프로젝트 타임라인 업데이트 |

#### 예시: Notion 자동 연동

```
회의 중 발언:
창업가: "재무 자료는 다음 주까지 보내드리겠습니다."

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Onno 자동 감지:

✅ 액션 아이템 감지
"재무 자료 요청"

Due: 12/9
담당: @김창업

→ Notion에 추가하시겠어요?
보드: [투자 심사 ▼]

[승인] [수정] [무시]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[승인] 클릭 시:
→ Notion에 자동 태스크 생성
→ 담당자에게 이메일 알림 (선택)
```

#### 설계 철학

> 워크플로우 연계는 대화 인텔리전스가 만들어낸 결과를
> **실제 실행까지 연결**해 주는 "AGI 스타일 편의 기능"
>
> **제품의 1차 가치**는 실시간 질문·인사이트·맥락 제공에 있음

### 3-6. Axis 3: 나만의 온노 (개인화 시스템)

#### 핵심 컨셉

> "나만의 온노" 컨셉은 **귀여운 캐릭터 꾸미기가 아니라**,
> 진짜로 **기능적으로 뭐가 달라지는지**를 명확히 하는 것

#### 3가지 개인화 메커니즘

**1. 도메인 레벨 시스템 (Lv.1-5)**

```
Lv.1 (시작)
- 기본 질문 제안 (모든 사용자 동일)
- 회의 전사 및 요약
- Notion 연동

↓ 100 XP (약 5회 회의)

Lv.2 (개인화 시작)
- 과거 맥락 자동 로드 ⭐
- 질문 톤 개인화 ("~하실까요?" vs "~인가요?")
- 선호 질문 타입 우선 제안

↓ 300 XP (약 15회 회의)

Lv.3 (전문가)
- 벤치마크 자동 비교 ⭐
- 리스크 자동 감지
- 사용자 패턴 인사이트

↓ 700 XP (약 35회 회의)

Lv.4 (마스터)
- 예측 질문 (다음 주제 예상) ⭐
- 패턴 기반 인사이트
- 고급 워크플로우 자동화

↓ 1500 XP (약 75회 회의)
```

### 3-7. UX: 플로팅 Always On Top 유틸리티

#### 설계 철학

Onno는 **화면 위에 떠 있는 작은 플로팅 창 형태**(Always On Top)로 제공됩니다.

```
┌─────────────────────────────────────┐
│  Zoom / Teams / Meet / Notion / ... │  ← 주 작업 화면
│                                     │
│              ┌──────────────┐       │
│              │  Onno 🎙️     │       │  ← 플로팅 창
│              ├──────────────┤       │     (항상 위)
│              │ 🔴 Critical   │       │
│              │ "LTV는?"      │       │
│              │ [사용] [무시] │       │
│              └──────────────┘       │
└─────────────────────────────────────┘
```

#### 장점

- **앱 전환 불필요**: 별도 창 전환 없이 바로 활용
- **방해 최소화**: 회의 흐름을 끊지 않음
- **빠른 액션**: 클릭 한 번으로 질문 사용

### 3-8. 회사별 커스텀 구축 (Productized Onboarding)

#### 설계 철학

각 기업·조직별로 회의 유형, 의사결정 방식, 전문 용어, 툴 환경이 다르기 때문에,

Onno는 **공통 엔진(SaaS)** 위에
**설정/템플릿 기반의 "프로덕트화된 커스텀"**을 제공합니다.

#### 온보딩 프로세스 (4단계)

```
Step 1: 회의 유형 정의
- 투자위원회, IR 미팅, 클라이언트 제안, 캠페인 리뷰 등

Step 2: 지식 베이스 연동
- 내부 문서·FAQ·용어 사전
- 과거 회의록 (선택)

Step 3: 템플릿 설정
- 회의 유형별 질문 템플릿
- 인사이트 규칙
- 요약·보고 포맷

Step 4: 시스템 연동
- CRM·PM·협업툴 (API 연동)
- 플러그인 설정
```

#### 차별점

- ❌ **인력을 갈아 넣는 SI 방식** (X)
- ✅ **설정·템플릿 위주의 셋업** (O)

→ 각 조직에 최적화되면서도 **빠르고 저렴한 도입** 가능

### 4-2. 도메인 특화 질문 그래프 및 질문 플레이북

#### 개념

스타트업 컴퍼니빌더 출신 CTO의 경험과
실제 투자/창업/컨설팅 현장의 **질문 패턴을 구조화**하여
"좋은 딜/프로젝트를 가려내는 질문 프레임"을 엔진에 반영

#### 질문 그래프 구조 (예시: VC 투자 심사)

```
[회의 단계 분류]
├─ Introduction (자기소개)
├─ Problem (문제 정의)
├─ Solution (솔루션 설명)
├─ Business Model (비즈니스 모델)
├─ Financials (재무 지표) ← 현재 단계
│  ├─ Revenue (매출)
│  │  └─ MRR 언급 → LTV 확인 필요 (Critical)
│  ├─ Unit Economics
│  │  └─ CAC 언급 → LTV:CAC 비율 확인 (Critical)
│  └─ Growth (성장률)
├─ Team (팀 구성)
└─ Ask (요청 사항)
```

#### 학습 메커니즘

각 조직별 회의 데이터가 쌓일수록:

1. **패턴 발견**: "우리 회사는 LTV:CAC를 중요하게 본다"
2. **플레이북 축적**: "우리 팀만의 질문 체크리스트"
3. **자동 적용**: 다음 회의부터 자동 반영

→ 플랫폼(MS/Zoom)이 쉽게 모방할 수 없는 **조직 특화 자산**
